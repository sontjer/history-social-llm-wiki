## 核心结论

1. **人物是历史类 Wiki 的链接枢纽**——必须双向贯通人物↔战役/战场/概念
2. **纯文件名匹配不够**——Wikipedia 文章用代词而非专名，必须补充强制关联链接
3. **繁体中文是个坑**——Obsidian 严格匹配字符，需要创建别名文件
4. **Obsidian vault 根目录**——必须选 `wiki/` 子目录而非库根目录
5. **采集工具要清洗**——`opencli wikipedia page` 输出含 CLI 垃圾，需要后处理
6. **自动分类需要持续维护**——新增人物时文件名关键词必须同步更新；短关键词可能误匹配，建议用全名/长名
7. **繁简双写是跨领域分类的关键**——zh.wikipedia.org 返回的标题简繁不定，classify() 必须同时匹配简繁写法
8. **中间名容忍**——中文译名含中间名（富兰克林·德拉诺·罗斯福），全名字串匹配会失败，应用片段匹配（姓名/姓）

## 快速上手

```bash
# 1. 采集 Wikipedia 文章
opencli wikipedia page "诺曼底登陆" --lang zh -f md > raw/诺曼底登陆.md

# 2. 清理 CLI 垃圾
sed -i '/Update available:/d; /^url: /d' raw/*.md

# 3. 编译到 wiki/
python3 scripts/wiki_init.py

# 4. 跨分类链接增强
python3 scripts/cross_link_wiki.py

# 5. 检测断链
for f in wiki/人物/*.md; do
  [ $(grep -c '战役/' "$f") -eq 0 ] && echo "❌ $f 缺战役链接"
done
```

## 新增：自增殖工作流（知识缺口驱动增长）

当资料库积累到 ≥30 篇后，可让已有资料自动发现"高频提及但未入库"的知识缺口，实现自我增殖。

### 原理

已入库的 Wikipedia extract 正文中含有大量实体名（人物/事件/机构/概念），统计这些实体在全文中的出现频次，与已有 wiki 文件名交叉比对——出现频次高但未入库的就是缺口。

### 四支柱模式匹配（通用，任何领域可用）

无需任何外部知识或预制清单，脚本自动提取文本中的命名实体：

| 支柱 | 匹配模式 | 示例 |
|------|---------|------|
| 人物 | `2-4汉字 + 天皇/首相/将军/大将` | 明治天皇、东条英机 |
| 事件 | `2-8汉字 + 战争/条约/运动/维新` | 甲午战争、明治维新 |
| 机构 | `2-6汉字 + 政府/军/省/部/局/会` | 关东军、陆军省 |
| 概念 | `2-8汉字 + 主义/论/化/观/思想` | 军国主义、脱亚论 |

### 工作流

```bash
# 1. 运行缺口探测（四支柱模式）
python3 脚本/gap_scanner.py /path/to/文库根

# 2. 按输出分级采集
#    ⭐ 高频（≥5次）：立即采集
#    ⭐⭐ 中频（3-4次）：按需采集

# 3. 采集到 raw/
opencli wikipedia page "实体名" --lang zh -f md > raw/实体名.md

# 4. 编译入库
python3 wiki_init.py                       # 分类编译
python3 脚本/cross_link_wiki.py /path/to/文库根  # 交叉链接

# 5. 重新跑一轮 gap_scanner 检查是否还有遗漏
```

### 实测数据

日本近代史研究库（62→119篇）：
- 4-pillar 四支柱模式扫描 → 发现 120 个候选 → 人工筛选出高频真实缺口
- 54 个实体有实际频率数据，31 个 ≥5 次
- 从采集到编译完成约 15 分钟


### 人物跳转与返回机制（黑红版 HTML）

每个 HTML 页面包含两层导航：
1. **阶段底部人物标签** — 各历史阶段下方的 `<div class="era-figures">` 包含关键人物名字
2. **点击标签 → 跳转到人物群像** — 每个标签带 `href="#person-XXX"`，直接定位到对应人物卡片
3. **返回按钮** — 每张人物卡片右上角有 `↑ 返回` 链接（hover 显示），使用 `javascript:history.back()` 精确回到阅读位置

**实现方式**：脚本自动为每个人物卡片添加 `id="person-{name}"` 锚点，并将 era-figures 中的纯文本 `<a>` 转换为带 `href` 的跳转链接。
