# 知识库使用与维护手册

> 生效日期：2026-07-12
> 适用范围：民国史 LLM Wiki（/mnt/webdav/ob-12400/Study of the National）

---

## 一、内容新增与更新规范

### 写作格式

- 每个新页面必须使用**双向链接**至少链到一个相关页面（`[[分类/页面名]]`），避免变成孤立笔记
- 撰写时使用**简单流畅的白话**写作风格
- 同一分类下的文档保持近似的章节结构即可，不强求统一模板

### 每次整理完成后的必做任务

每次新增或更新内容后，必须按顺序执行以下步骤：

```
1. python3 wiki_init.py          # 重新编译（关联文档段使用 [[xxx]] 无前缀格式）
2. python3 cross_link_wiki.py    # 更新跨分类反向链接（本页提及段使用 [[分类/xxx]] 有前缀格式）
3. python3 check_links.py        # 检查双向链接完整性
4. 手动更新「📑索引/00_全目录.md」   # 反映知识库最新内容
5. 前往「log.md」添加一条记录        # 写清楚本次改了什么
```

### 双向链接最佳实践

正确的链接格式是 Obsidian 关系图谱正常显示的关键：

```
## 关联文档                    ← wiki_init.py 生成，同分类
- [[孙中山]]                   ✅ 无分类前缀，Obsidian 全库搜索自动匹配
- [[蒋介石]]

## 本页提及的相关条目           ← cross_link_wiki.py 生成，跨分类
- [[事件/辛亥革命]]              ✅ 有分类前缀，跨分类区分同名实体
```

**重要规则：**
- 关联文档段落**绝对不要**使用 `[[人物/孙中山]]` 带分类前缀格式
- 人物是链接枢纽——每篇人物必须有到事件的链接，每篇事件必须有到人物的链接
- 自动扫描不能覆盖的跨分类关系，需人工强制补链到 `raw/` 源文件
- 强制补链需使用 curl WebDAV PUT（davfs2 FUSE mount 写入不可靠）

## 二、回答提问的要点

### 回答前

先根据「📑索引/00_全目录.md」找出相关页面，再根据那些页面的内容来回答。

### 回答后

如果问答中有值得保留的信息，建议存到知识库，但存储前先询问用户。

---

## 三、知识库维护检查要点

检查 ① 失效双向链接 — 搜索 [[ 语法，验证每个指向的 .md 文件是否存在
检查 ② 孤立页面 — 对比哪些页面从未被其他页面的 [[wikilink]] 指向
检查 ③ 重复或高度相似页面 — 繁简、别名
检查 ④ 内容矛盾 — 跨分类引用同一事件说法不一致
检查 ⑤ 产出报告
