# 人物核心链接原则

> 历史、社科类 LLM Wiki 的链接设计铁律

---

## 为什么人物是枢纽

在历史知识库中，人物是整个网络的**中枢节点**：
- 战役由人物指挥或参与
- 政策/概念由人物提出或推动
- 部队由人物统帅
- 时间线按人物活动划分

每个浏览路径必然经过人物：
```
时间线 → 人物 → 战役 → 人物 → 概念
  ↓                          ↑
  └───────── 战场 ───────────┘
```

## 链接标准

| 方向 | 要求 | 检测方法 |
|------|------|---------|
| 战役→人物 | 每篇战役必须链到相关人物 | `grep -c '人物/'` > 0 |
| 人物→战役 | 每个人物必须链到相关战役 | `grep -c '战役/'` > 0 |
| 人物→部队 | 统率关系 | 自动 + 人工补 |
| 人物→概念 | 提出/推动/关联 | 自动 + 人工补 |

## 为什么自动匹配不够

`cross_link_wiki.py` 通过文件名精确匹配，但历史文本常用：
- 代词：「他率军北上」
- 称谓：「委员长」、「总统」、「元首」
- 官职：「盟军最高司令」
- 绰号：「隆美尔」=「沙漠之狐」，但文本里只写「沙漠之狐」
