## 一、内容新增与更新规范

### 写作格式

- 每个新页面必须使用**双向链接**至少链到一个相关页面（`[[分类/页面名]]`），避免变成孤立笔记
- 撰写时请使用**简单流畅的白话**写作风格，不要使用文言文或艰深词汇
- 同一分类下的文档保持近似的章节结构即可，不强求统一模板

### 每次整理完成后的必做任务

每次新增或更新内容后，必须按顺序执行以下步骤：

```
1. python3 wiki_init.py          # 重新编译（含同分类关联文档）
2. 手动更新「📑索引/00_全目录.md」   # 反映知识库最新内容
3. 前往「log.md」添加一条记录        # 写清楚本次改了什么
```

## 二、回答提问的要点

### 回答前

当我向你提问时，请先根据「📑索引/00_全目录.md」找出相关页面，再根据那些页面的内容来回答。

### 回答时

在你的回复中请清楚标明你引用了哪些页面的信息，让我能知道答案的依据来源。

### 回答后

如果我们的问答中有值得保留的信息，你可以建议我将这些信息存到知识库，但存储前请先询问我。

## 三、知识库维护检查要点

如果我要求你对知识库进行维护或检查，请执行以下五项检查并输出报告。

### 检查项目

```
检查 ① 失效双向链接
   在 wiki/ 内搜索 [[ 语法，
   逐一验证每个 [[分类/页面名]] 指向的 .md 文件是否存在。
   找到断链就记录路径和断链名称。

检查 ② 孤立页面
   扫描 wiki/ 下所有 .md 文件，
   对比哪些页面从未被任何其他页面的 [[wikilink]] 指向。
   孤立页面 = 在全文中搜索不到被链接的页面。
   找到就记录路径。

检查 ③ 重复或高度相似页面
   比对文件名（去掉 .md），找出命名相似或同义的页面。
   中文关键词：别名 / 繁简 / 全称缩写
   如发现，提出合并建议但不自动执行。

检查 ④ 内容矛盾
   这个检查依赖人工阅读判断，我能做的是：
   挑出跨分类引用同一事件但说法可能不一致的页面。
   标记出来供你判断。

检查 ⑤ 产出报告
   格式如下（markdown）：
```

### 报告模板

```markdown
## 知识库维护报告 YYYY-MM-DD

### ① 失效链接：N 处
页面 | 失效链接 | 建议
-----|---------|-----

### ② 孤立页面：N 篇
路径 | 建议
-----|-----

### ③ 重复/相似页面：N 组
页面 A | 页面 B | 建议
------|-------|-----

### ④ 内容矛盾：N 处
页面 | 疑虑 | 说明
-----|------|----

### ⑤ 总评
一句话总结知识库健康状况。
```
