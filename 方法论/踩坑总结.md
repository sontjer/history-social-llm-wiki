# 历史、社科类 LLM Wiki 踩坑总结

> 实战于抗日战争库（67篇）和二战库（82篇）后的经验提炼
> 2026-07-10

---

## 坑 ① Obsidian vault 根目录选错 → 关系图谱空白

**现象：** 所有 `[[分类/页面名]]` 链接在 Obsidian 关系中不显示任何连线。

**根因：** `[[战役/诺曼底登陆]]` 这种链接从 vault 根目录开始解析。如果在 Obsidian 中打开的是库根目录（如 `Study of War of Anti-Japan/`），而非 `wiki/` 子目录，则解析路径变为 `Study of War of Anti-Japan/战役/诺曼底登陆.md`，但实际文件在 `wiki/战役/` 下。

**修复：** Obsidian 中 vault 根目录必须指向 `wiki/` 子目录。

**检测：** 点击 `[[战役/xxx]]` 链接，Obsidian 提示"无法解析文件"即为该问题。

## 坑 ② Wikipedia 简繁标题不匹配 → 死链接

**现象：** 部分 `[[人物/蒋介石]]` 链接显示为红色（找不到目标文件）。

**根因：** `opencli wikipedia page --lang zh` 返回的页面标题可能使用繁体中文（如「蔣中正」），保存的文件名也是繁体。但在战役文档中手动补充链接时用了简体「蒋介石」。Obsidian 严格匹配字符，简繁不一致 = 死链接。

**影响范围：** 抗日战争库中 3 个人物受影响：蒋介石/孫立人/陳誠

**修复：** 
```bash
# 创建别名文件（简单复制即可，WebDAV 不支持 symlink）
cp wiki/人物/蔣中正.md wiki/人物/蒋介石.md
cp wiki/人物/孫立人.md wiki/人物/孙立人.md
```

**检测：** 
```bash
grep -oP '\[\[\K[^]]+' wiki/**/*.md | while read link; do
  if echo "$link" | grep -q '/'; then
    cat="${link%%/*}"; target="${link#*/}"; target="${target%|*}"
    [ ! -f "wiki/$cat/$target.md" ] && echo "❌ $cat/$target.md 缺失"
  fi
done
```

## 坑 ③ opencli 输出污染源文件

**现象：** 源文件末尾包含非文章内容：
```
Update available: v1.8.4 → v1.8.6
Run: npm install -g @jackwener/opencli
url: https://zh.wikipedia.org/wiki/...
```

**根因：** `opencli wikipedia page --lang zh -f md > file.md` 将 stdout 全部重定向，
其中包括 CLI 工具的更新提示和版本信息。

**影响：** 这些垃圾行可能干扰 Obsidian 的 Markdown 解析器，
导致其后的 `[[wikilink]]` 不被识别。

**修复：**
```bash
sed -i '/Update available:/d' raw/*.md
sed -i '/Run: npm install/d' raw/*.md
sed -i '/^url: /d' raw/*.md
```

**预防：** 将清洗步骤加入采集工作流，每次采集后自动执行。

## 坑 ④ 纯文件名匹配不够 → 人物↔战役断裂

**现象：** 运行 `cross_link_wiki.py` 后，许多人物文档仍无战役链接，
战役文档也没有人物链接。

**根因：** `cross_link_wiki.py` 扫描文档全文，用**文件名**做精确匹配。
但 Wikipedia 文章常用代词和称谓：
- "盟军司令下达总攻令" → 不会匹配「艾森豪威尔」
- "他指挥了这场关键战役" → 不会匹配「诺曼底登陆」
- "日军统帅部决定..." → 不会匹配「东条英机」

**影响范围：** 二战库 20 个人物中有 10 人零跨分类链接。

**修复：**
1. 先跑 `cross_link_wiki.py` 自动扫描
2. 执行断链检测脚本
3. 对零链接人物**人工补充强制历史关联链接**

```bash
# 对零链接人物补充强制关联
echo -e "\n---\n## 本页提及的相关条目\n- [[战役/xxx]]\n" >> wiki/人物/xxx.md
```

**强制链接添加规范：**
- 追加在已有 `## 本页提及的相关条目` 段落内
- 只链向确实相关的条目，不滥链
- 格式统一：`- [[分类/页面名]]`
- 双向补充：战役→人物 + 人物→战役

## 坑 ⑤ 源文件尾部 `---` 分隔符干扰 wikilink

**现象：** `[[wikilink]]` 在文档中存在，但 Obsidian 不解析。

**根因：** 源文件 (或编译后的 wiki 副本) 末尾存在：
```

> 来源：https://...

---
## 关联文档
```

被 `---` 包围的段落可能被 Obsidian 当作 frontmatter 区块，
导致其后的 `[[wikilink]]` 不被解析为有效链接。

**修复：** 统一为无分隔的干净格式：
```
> 来源：https://...
## 关联文档
- [[人物/xxx]]
```

## 总检查清单

每次建库或扩充分类后，按序执行：

```bash
# 1. 清洗源文件
sed -i '/Update available:/d; /Run: npm install/d; /^url: /d' raw/*.md

# 2. 编译
python3 wiki_init.py

# 3. 自动跨分类链接
python3 cross_link_wiki.py

# 4. 检测战役→人物断裂
for f in wiki/战役/*.md; do
  [ $(grep -c '人物/' "$f") -eq 0 ] && echo "❌ $f 无人"
done

# 5. 检测人物→其他分类断裂
for f in wiki/人物/*.md; do
  cross=$(grep -cE '战役/|概念与政策/|部队编制/|战场/|时间线/' "$f")
  [ "$cross" -eq 0 ] && echo "❌ $f 零跨分类链接"
done

# 6. 检测死链接
grep -oP '\[\[\K[^]]+' wiki/**/*.md | while IFS= read -r link; do
  if echo "$link" | grep -q '/'; then
    cat="${link%%/*}"; target="${link#*/}"; target="${target%|*}"
    [ -d "wiki/$cat" ] && [ ! -f "wiki/$cat/$target.md" ] && echo "❌ $target"
  fi
done

# 7. 人工补强制链接
# 对零链接人物按历史知识补充
```
