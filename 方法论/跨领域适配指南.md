# 跨领域分类适配指南

从抗日战争库到美国史库——classify() 函数跨领域适配的实战方法论。

## 核心原则

1. **分类体系随主题变化**：军事史保留「部队编制」「装备」，政治史换「机构」「概念」，外交史换「条约」「主义」
2. **繁简双写不可跳步**：zh.wikipedia.org 标题简繁不定，每个中文关键词必须有简繁两版
3. **中间名容忍**：西方人物译名含中间名，全名匹配会失败，应改用姓/名的片段匹配

## 通用 classify() 优先级

```
时间线 → 事件/战役 → 人物 → 机构 → 概念与政策（兜底）
```

## 繁简双写铁律

| 简体 | 繁体 | 说明 |
|------|------|------|
| 战争 | 戰爭 | 缺繁体会漏 美國獨立戰爭 |
| 运动 | 運動 | 缺繁体会漏 民權運動 |
| 政策 | — | 不含繁简差异但常含特殊字符 |

## 兜底陷阱

「概念与政策」是 classify() 的默认兜底。以下情况会误入：
- 新人物文件名不匹配人物关键词
- 繁简字符不匹配
- 文件名含特殊字符（空格、括号、点）

每次新增条目后必须检查兜底分类，将误入文件移至正确目录。
